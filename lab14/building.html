Building the Homelab

This page explains how I built my homelab from the hardware on my desk to the software running virtual machines and storage.

Hardware

When starting a homelab, the first decision is choosing hardware. I built a small desktop-style server that balances performance, cost, and power usage.

  • ASRock motherboard with support for future upgrades.
  • Intel Core i3 processor.
  • 16 GB of RAM (with room to add more later).
  • Fast SSD for the operating system and virtual machines.
  • Larger hard drive for bulk storage and backups.

Networking

Networking is the backbone of any homelab. Properly configured switches, routers, and wireless access points keep devices connected and secure.

  • Separate network for lab devices and personal devices.
  • Basic firewall rules to control traffic between segments.
  • Ethernet connections for servers and key devices.
  • Simple monitoring tools to watch network usage.

Software Setup

The software stack is where most of the experimenting happens. I use a mix of virtualization and storage tools to run the lab.

  • Linux installed as the base operating system.
  • A hypervisor to manage virtual machines.
  • TrueNAS or similar software for file storage.
  • Docker containers for lightweight services.
  • Regular backups so I can recover from mistakes.

System Architecture

The homelab connects a single physical server to several virtual machines, storage pools, and the rest of the home network.

Central host connected to VMs, storage, and network devices.
